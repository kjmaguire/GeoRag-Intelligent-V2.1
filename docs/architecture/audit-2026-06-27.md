## 3. Critical findings (🔴)

### C1 — Qwen3 dimension regression in the re-index/init path (T1)
**Confirmed (live + 2 agents + lead).** The live `georag_chunks` collection is
correctly **1024-dim**, but every path that could *recreate* it still declares
**384-dim**:
- `src/dagster/georag_dagster/assets/index_document_passages.py:67` `EMBED_DIMENSIONS = 384` → `VectorParams` at :263
- `index_reports.py:64` (384) and `index_public_geoscience.py:82` (384)
- `init_qdrant.py` hardcodes 384, reads no dimension env, has no `--recreate`/guard
- Dagster guard only checks the **sparse** slot, not the dense dim.

**Impact:** re-materializing any index asset (a routine Dagster op) silently
recreates the collection at 384-dim → all 1024-dim writes 400, retrieval
returns nothing or garbage. This is the exact failure class as the
2026-06-01 `georag_chunks` outage.
**Fix:** read `settings.EMBEDDING_DIMENSION` (1024) everywhere; add a
fail-fast guard that refuses to create/shrink an existing collection to a
smaller dim. Verified live: collection size = 1024 (so the bomb hasn't gone
off — it's armed).

### C2 — Laravel RLS GUC leaks across requests under PgBouncer (T2)
**Confirmed (lead read of source).**
`app/Support/SetsWorkspaceRlsContext.php:44` uses
`set_config('app.workspace_id', ?, false)` — the `false` makes it
**session-scoped, not transaction-scoped**. Laravel connects through
`pgbouncer:6432` (transaction-mode pooling). The GUC therefore persists on the
pooled backend connection *after* the request's transaction ends, with no
`SET LOCAL` and no `DISCARD ALL`. The next request that reuses that backend
inherits the previous tenant's `app.workspace_id`.
The class docstring (lines 21-25) even **claims it uses `SET LOCAL`** — it does
not (comment/code drift on top of the bug). A sibling instance exists in
`emit_audit` (session GUC on a pooled connection without a transaction, medium).
**Impact:** cross-tenant data exposure once >1 workspace exists. Latent today
(single workspace).
**Fix:** wrap controller bodies in a transaction and use `SET LOCAL`
(`set_config(..., true)`), or reset the GUC at request end. Add a PgBouncer
`server_reset_query = DISCARD ALL` defense-in-depth (note: only safe in
session pooling) — preferred fix is transaction-scoped binding.

### C3 — `search_documents` fails open to the default tenant (T2)
**Confirmed (lead + 2 agents — retrieval & agentic).**
`src/fastapi/app/agent/tools.py:1678` seeds `_workspace_id =
"a0000000-…-001"` and derives the real workspace via a per-call DB lookup
(:1681-1687); on **any** exception it keeps the default (:1688-1691). The
mandatory Qdrant `hybrid_query` workspace filter then fences to the *default*
tenant's documents.
**Impact:** a failed/missing lookup silently serves the seed tenant's chunks to
the caller — wrong-tenant exposure + wrong answers in the RAG hot path. The
dedicated guard test for this is red/unrun. Especially insidious because the
default *is* the only real tenant today, masking the bug.
**Fix:** fail **closed** (raise/empty) on lookup failure; take `workspace_id`
from the authenticated JWT claim (the acknowledged "Module 9" debt at
tools.py:1675-1677), not a DB lookup defaulting to tenant 1.

### C4 — (cross-listed) Dagster/runtime dim mismatch into one collection
Reported independently by the `ingest-structured` agent as a distinct critical;
it is the same root cause as **C1** (writer 1024 vs index-asset 384 into
`georag_chunks`). Tracked once under C1.

## 4. High findings (🟠) — grouped by theme

### T1 — Qwen3 swap half-applied
- **Reranker swap never reached prod.** Live `.env RERANKER_MODEL_PATH` is
  empty → production runs **bge-reranker-base**, not Qwen3-Reranker-0.6B as the
  manual (Ch08/Ch18) states. The reranker sidecar then reports a *stale
  version* to the audit trail (medium). → wire it or correct the manual.
- **Query/corpus embedding asymmetry** (found by 3 agents). `tools.py:1655`
  hardcodes the bge-era instruction prefix *"Represent this geological query
  for searching relevant passages: …"* on the query side, ignores
  `EMBEDDING_QUERY_PROMPT_NAME`, and doesn't normalize at index time. After the
  Qwen3-Embedding swap this prefix is wrong for the model → measurable nDCG
  loss. → use the Qwen3 query prompt; align index/query normalization.
- **No startup dimension-parity guard** despite a `config.py` comment claiming
  fail-fast exists; embedding sidecar default is still 384-dim bge-small.

### T2 — Tenant-isolation fail-open seams (all latent; single-tenant today)
- **`/pdf/ocr_region` never binds `app.workspace_id`** → RLS-FORCED
  `silver.pdf_ocr_results` reads/writes error or silently return nothing.
- **Streaming JWT omits `workspace_id`** on the main query path → the
  lifecycle/RLS guard is silently skipped (laravel-bridge).
- **`conversation_id` IDOR** → unauthorized chat-history is pulled into the LLM
  prompt (cross-tenant chat-history exfil; laravel-bridge, medium-high).
- **Lead-found IND-1:** `deps.py:154` binds `app.workspace_id` only *inside*
  `if project_id` → workspace-scoped, project-less queries (cross-project chat)
  skip the bind and hit the `IS NULL` fail-open hatch. (`nodes.py:2018` sets
  `project_id=None`, confirming the trigger exists.)

### T3 — Hallucination/citation enforcement bypassed on agentic path
- **Agentic V2 discards `should_retry`** → Layers 3/4/6 *detect* fabrication
  but never reject or retry (the `ModelRetry` loop is dead on this path).
- **Layer 2 typed-output validation never runs on the agentic path** → orphan
  citation markers + empty grounding slip through (medium).
- **Citation marker style mismatch:** prompt emits colon-form `[REF:1]` but
  Layers 2/3 regex only match dash-form `[REF-1]`, with no normalization → the
  validators see *zero* markers on the agentic path.
- **`assemble_response` fabricates a `[DATA-1]` citation** for ungrounded
  answers and the agentic path commits it as valid (medium). Combined with the
  lead's IND-6: numeric claims are hard-enforced, but a qualitative ungrounded
  answer can reach the user on a placeholder citation — a Hard-Rule-4 seam.
- **Layer 4 entity grounding is bypassable** (medium): every 3+char tool-result
  token joins the grounded bag, so fabricated names that coincide with any
  token pass.

### T4 — §04p PDF persistence & async safety
- **`transactional_workspace_session` has a malformed `str()` call** → raises
  `TypeError` on *every* invocation, silently disabling all §04p dual-write
  persistence. (High — verify against live ingest.)
- **Scanned-PDF OCR fallback renders the whole document to RAM at 250 DPI** →
  unbounded memory on large scanned reports (ties to the known §04p
  subprocess-pool instability).
- **Blocking boto3 + sync HTTP-read inside the asyncio event loop** in the
  persist task (Hard Rule 2 violation; medium). Also in `ingest_zip_archive`
  (low/uncertain).

### T5 — Database operational time-bombs
- **pg_partman maintenance is never scheduled** → `audit_ledger` /
  `workflow_runs` partitions exhaust ~3 months after deploy (writes start
  failing). → schedule `partman.run_maintenance_proc()`.
- **Init-script ordering trap:** on a fresh volume, postgis-dependent
  extensions run *before* PostGIS is installed; the ordering comments are
  backwards. → renumber; matches the recurring init-ordering theme.
- **Lead IND-7: GPU at 97% VRAM** (`nvidia-smi`: 469 MiB free of 20 GB). The
  `0.72` main + `0.46` VL gpu-mem-util split + sidecars leave near-zero
  headroom → OOM on reload/fragmentation/long-seq. Explains why context is
  served at **8K not the planned 16K** (lead IND-8). → alert on free<1GB; run
  vllm-vl on-demand.

### T6 — Frontend
- **Stored XSS in every MapLibre popup**: DB feature properties are
  interpolated raw into `Popup.setHTML()`. Tainted ingested data → script
  execution in the geologist's browser. → escape/sanitize all popup HTML.
- **37 files carry blanket `@ts-nocheck`** (incl. the core Chat RAG surface and
  auth pages), silently defeating the tsc CI gate — directly undercuts today's
  "tsc gate fixed" work (medium). → burn down `@ts-nocheck`; CI should count
  them.

### T2/T7 — Security posture
- **Ingested document content reaches the LLM context unsanitized and
  undelimited** (prompt-injection) — no instruction/data separation. → wrap
  retrieved chunks in clear data delimiters; add injection heuristics.
- **`SECURITY.md` is stale and asserts the opposite of the live enforcement
  default** (T7). → reconcile.

## 10. Method & provenance

- **Loop process** (per the two Anthropic references): observe (recon: MANUAL,
  CLAUDE.md, live container/DB/Qdrant state) → plan (14-subsystem
  decomposition) → act (expert fan-out + independent live checks) → reflect
  (per-finding adversarial verification + lead cross-check) → repeat.
- **Fan-out:** 14 domain-expert finders (security/perf/correctness/maintain),
  wave-chunked (2 finders/wave, 5× retry) after an initial 14-wide launch hit a
  server burst rate-limit. 85 agents total, 6.46M tokens, ~54 min.
- **Adversarial verification:** every finding re-checked by an independent
  skeptic against current code (default to refute). This downgraded 1 finding
  and refuted 5 — working as intended.
- **Independent self-check:** lead reviewer verified the highest-risk items
  against live systems (`docker exec psql`, Qdrant REST, `nvidia-smi`) — the
  source of C2/C3 confirmation and IND-1/2/3/5/7/8.
- **Working artifacts:** `tmp/audit-2026-06-27-independent-scratch.md` (lead
  findings), `tmp/audit-2026-06-27-workflow-digest.md` (full agent findings
  with verifier verdicts). Full raw result in the task output file.
