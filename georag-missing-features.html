GeoRAG — Missing Features & Gap Analysis

Companion to georag-architecture.html · Production-readiness gap inventory · v1 · 2026-05-07
📋 HOW TO READ THIS DOC
This document inventories features and capabilities not yet present in the GeoRAG codebase as of 2026-05-07. The companion georag-architecture.html describes what the system is; this doc describes what's missing.

Severity ladder:

  • P0 Blocks pilot. Must land before the first paying customer touches the system.
  • P1 Blocks GA. Acceptable for a friendly pilot but mandatory for a public launch.
  • P2 Blocks scale. Needed for the second / third tenant or once usage crosses tier-2 jurisdictions.
  • P3 Polish. Quality-of-life / contributor-experience / nice-to-have. Defer until the above is done.
  • PARTIAL Scaffolded but not finished — config exists, no implementation; or implemented for one path, missing on others.
  • DONE Closed during the 2026-05-07 production-readiness session.

Scope: covers the entire stack — frontend, application layer, FastAPI, datastores, ops, observability, customer-facing, compliance, docs. Items already done in the 2026-05-07 session are listed for completeness with the DONE badge.

Reading order: §00 is the one-glance dashboard. §15 is the prioritised roadmap. Sections in between are deep dives by domain — read the ones you own.
