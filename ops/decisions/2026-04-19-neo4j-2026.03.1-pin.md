# Decision: Neo4j 2026.03.1 Accepted as V1 Pin

**Date:** 2026-04-19
**Authorized by:** Kyle Maguire
**Recorded by:** devops-engineer agent

---

## Decision

Neo4j **2026.03.1** Community Edition is accepted as the effective V1 pin for GeoRAG. The digest `sha256:a5feb81d916c82d09186807ee8f8a523eb430d578fa6015f37ae72a07f976537` committed in Phase B of Module 1 infrastructure work remains in place and is the authoritative lock.

## Reason

The tag `neo4j:2026.02.3-community` referenced in the architecture doc (Section 12) does not exist in the Docker Hub registry and never did. When Docker resolved `neo4j:2026.02.3-community` it silently fell back to the floating tag `neo4j:2026-community`, which resolved to version **2026.03.1** as of 2026-04-19. Phase B Fix 1 (finding IMG-01) replaced the non-existent tag with a digest pin at the 2026.03.1 build. There is no 2026.02.3-community image available in the public registry to revert to.

## Current compose reference

```yaml
image: neo4j:2026-community@sha256:a5feb81d916c82d09186807ee8f8a523eb430d578fa6015f37ae72a07f976537
```

Both `neo4j` and `neo4j-warmup` services use this digest.

## Follow-up task — Module 10 doc sweep (v1.9 -> v1.10)

The architecture doc `georag-architecture.html` Section 12 must be updated to reflect 2026.03.1 as the V1 pin. This is a documentation-only change. No code or compose changes are required at that time. Assign this update to the Module 10 architecture doc sweep task.

Specific text to update: any reference to `neo4j:2026.02.3-community` should become `neo4j:2026.03.1-community` (or note the digest pin as the authoritative lock).

## Impact assessment

- No functional regression. Neo4j 2026.03.1 CE is a patch release over 2026.02.x. Community Edition data store format is forward-compatible; existing `neo4j_data` volume opened cleanly after the digest change.
- APOC plugin version compatibility should be re-verified against 2026.03.1 if it has not been confirmed post-Phase B recreate.
- No Enterprise features were being used. CE constraints (no clustering, no active warmup, no database-level RBAC) remain unchanged.
